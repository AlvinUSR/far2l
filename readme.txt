Система "TREX"
Главная посылка - "ВСЁ ЕСТЬ КОНЕЧНЫЙ АВТОМАТ"

Компоненты:
1. tmaster - мастер-процесс (организация взаимодействия между компонентами)
2. tserver - организация взаимодействия с внешними сущностями (брокеры, клиенты, внешние подключаемые компоненты)
2. tcore - вычислительное ядро (анализрыночных данных, вычисления для постоения графиков и индикаторов, торговые алгоритмы)
3. tdb - субд (хранение и предоставление данных, необходимых компонентам системы)
4. tterm - пользовательский терминал (графический - для пользователя и администратора, администраторский - для администратора)  
5. tlog - менеджер системы логирования

Общие библиотеки:
- определение общих типов (types)
_____
- базовые определения (base)
    -- система обработки ошибок и исключений (error handling system - ehs)
    -- описание процесса системы (process)
        --- инициализация
            ---- парсинг аргументов командной строки
            ---- применение настроек
            ---- создание необходимых служб
            ---- запуск служб
        --- работа
        --- завершение
            ---- сохранение необходимых данных
            ---- остановка служб
    -- описание службы процесса (service)
        --- инициализация
        --- работа
        --- завершение
_____
- связь между компонентами (components communication system - ccs)
    -- универсальное решение, которое позволяет менять принцып взаимодействия между компонентами в процессе развёртывания системы
        --- sockets
        --- shared memory
    -- запросы и данные передаются в формате JSON
_____
- пул потоков (thread pool - thp)
- система событий (events system - evs)
- конечные атоматы (final state machine - fsm)

_____
- парсеры (parsers - prs)
    -- JSON

_____
- управление настройками (settings managment system - sms)
_____
- фабрика объектов (object factory - fac)
_____
- система логирования (log)
